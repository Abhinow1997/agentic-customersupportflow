Agentic Returns: Actor-Critic Architecture

This diagram maps the Python/FastAPI backend workflow to a Reinforcement Learning (RL) paradigm. The system acts as a Markov Decision Process where the goal is to reach a terminal state with maximum confidence while avoiding negative reward penalties (human intervention).

1. State (S_t)

Before the AI makes decisions, the system gathers deterministic data (Snowflake DB, native date math). This grounds the LLM, preventing hallucinations and setting a perfect initial observation space.

2. Actor (Policy/Agent)

The Researcher Agent acts as the RL Actor. It observes the State and takes an Action: hypothesizing an answer to one of the 5 Critical Questions based on context.

3. Critic (Environment)

The Policy Agent acts as the Critic. It queries real Walmart policies (RAG) to validate the Actor's hypothesis. It provides the Reward Signal: +0.1 confidence boost for passing, -20% confidence penalty for failing.

4. Optimization & Penalty

If the Critic flags insufficient data, the workflow pauses for human follow-up. In an RL context, human intervention incurs a severe penalty. Over time, Policy Gradients learn to minimize this path.
